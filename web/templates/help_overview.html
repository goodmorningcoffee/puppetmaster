How PUPPETMASTER Works
WHAT IS THIS TOOL?

PUPPETMASTER analyzes SpiderFoot OSINT scan data to identify "sock puppet" networks — groups of websites that appear independent but are actually controlled by the same entity.

COMPLETE WORKFLOW

Step 1: DISCOVER — Find domains to investigate

  • Use option [1] to scrape domains from keywords
  • Or use option [2] to load your own domain list

Step 2: SCAN — Run SpiderFoot on those domains

  • Use option [3] to batch scan all domains
  • Scans run in parallel (configurable)
  • Progress is saved — you can resume if interrupted

Step 3: ANALYZE — Find connections between domains

  • Use option [5] to analyze the SpiderFoot exports
  • PUPPETMASTER finds shared infrastructure (analytics IDs, certs, contacts...)

Step 4: REVIEW — Examine the findings

  • Use option [6] to view previous results
  • Drill down into clusters of connected domains
THE SIGNAL HIERARCHY

PUPPETMASTER classifies evidence into three tiers:

  • SMOKING_GUN — one match = definitive connection (Google Analytics IDs, AdSense pubs, SSL fingerprints, unique emails...)
  • STRONG — 2+ matches = likely connection (custom nameservers, IPs, WHOIS, phone numbers, crypto addresses...)
  • WEAK — context only, not evidence (CDNs, hosting, common libraries)

One shared unique identifier = same operator.
